# 오디오 파일 & 폴더 규칙 브리핑

## 1. 네팔어 코드 통일: "np"

프로젝트 전체에서 네팔어 코드는 **np**로 통일한다.

| 항목 | 올바른 값 | 잘못된 값 |
|------|-----------|-----------|
| 폴더 경로 | audio/**np**/ | audio/~~ne~~/ |
| JSON 설명 키 | "**np**": "..." | "~~ne~~": "..." |
| JSON 오디오 키 | "audio_**np**": "..." | "audio_~~ne~~": "..." |
| 파일명 | E01-C02_01_**np**.mp3 | E01-C02_01_~~ne~~.mp3 |

- ne (ISO 언어코드)와 np (국가코드) 중 **np로 통일**
- "ne" 폴더가 생기면 즉시 "np"로 변경할 것

## 3. 파일 네이밍 규칙

### 자막 설명 오디오
```
{클립ID}_{자막번호2자리}_{언어}.mp3
```
- 한국어: `E01-C02_01_kr.mp3`
- 네팔어: `E01-C02_01_np.mp3`

### 퀴즈 오디오
```
{클립ID}_Q{퀴즈번호2자리}_{언어}.mp3
```
- 한국어: `E01-C02_Q01_kr.mp3`
- 네팔어: `E01-C02_Q01_np.mp3`

### ElevenLabs 업로드 시 이름 (확장자 제외)
- `E01-C02_01_kr` (ElevenLabs가 자동으로 .mp3 붙임)
- `E01-C02_01_np`

## 5. 새 클립 작업 시 필수 체크리스트

새 클립(예: E01-C03) 작업을 시작할 때 반드시 확인:

### JSON 체크
- [ ] `youtube_id`가 채워져 있는가 (비어있으면 영상 안 나옴)
- [ ] `clip_id`가 정확한가

### 폴더 체크
- [ ] `audio/kr/{클립ID}/` 폴더가 존재하는가
- [ ] `audio/np/{클립ID}/` 폴더가 존재하는가
- [ ] 폴더가 비어있으면 최소 `.gitkeep` 파일이라도 있는가

### 오디오 파일 체크
- [ ] kr 파일과 np 파일 개수가 일치하는가
- [ ] 파일명이 `{클립ID}_{번호}_{언어}.mp3` 형식인가
- [ ] `ne`가 아닌 `np`로 되어있는가

## 6. 과거 사고 기록 & 교훈

### 사고 1: ne → np 폴더 이름 변경 시 파일 소실
- **상황**: `audio/ne/` 를 `audio/np/`로 rename 할 때 E01-C02 mp3 10개 삭제됨
- **원인**: git rename 과정에서 중첩 폴더 처리 오류
- **교훈**: 폴더 이름 변경 시 파일 백업 먼저. 변경 후 파일 개수 반드시 확인

### 사고 2: 빈 오디오 폴더 → 유튜브까지 안 나옴
- **상황**: `audio/np/E01-C02/`가 비어있으니 네팔어 음성뿐 아니라 유튜브 영상까지 로드 실패
- **원인**: learn.html에서 오디오 경로 에러가 발생하면 이후 코드(유튜브 초기화)도 멈춤
- **교훈**: 오디오 파일이 없어도 에러가 전파되지 않도록 try-catch 필요. 폴더는 항상 미리 생성

### 사고 3: youtube_id 누락
- **상황**: E01-C02.json의 youtube_id가 빈 문자열이어서 영상 안 나옴
- **교훈**: 새 클립 JSON 만들 때 youtube_id를 Master Map에서 반드시 복사해 넣을 것

### 사고 4: learn.html 오디오 재생 에러 전파 (수정 완료)
- **상황**: learn.html line 1007에서 `explainAudio.play()`에 try-catch 없이 호출 → mp3 파일 없으면 uncaught promise error 발생 → 유튜브 플레이어 초기화까지 차단됨
- **수정**: `.play()` 호출을 `try { explainAudio.play().catch() }` 로 감싸서 에러 전파 차단
- **교훈**: `new Audio()`와 `.play()`는 반드시 에러 처리 필요. 오디오 로드 실패가 다른 기능(유튜브, UI 등)에 영향 주면 안 됨. 향후 오디오 관련 코드 추가 시 항상 try-catch + .catch() 패턴 사용할 것

## 7. 캐시 방지 규칙

- JSON/JS 로드 시 버전 쿼리스트링 사용
- 형식: `?v=YYYYMMDD` 또는 `?v=YYYYMMDDx` (같은 날 여러 번이면 a, b, c 추가)
- 현재 버전: `?v=20260303`
- **새 JSON 추가하거나 수정할 때마다 버전 업데이트 필수**

```javascript
fetch('data/' + DRAMA + '/' + CLIP_ID + '.json?v=20260303')
fetch('data/' + DRAMA + '/episodes.json?v=20260303')
```

## 8. 배포 전 최종 확인

커밋 & 푸시 전에 반드시:
1. JSON의 youtube_id 확인
2. audio/kr/{클립ID}/ 에 kr mp3 파일 존재 확인
3. audio/np/{클립ID}/ 에 np mp3 파일 존재 확인 (ne 아님!)
4. learn.html 캐시 버전 업데이트 여부 확인
5. learn.html 오디오 재생 코드에 try-catch 있는지 확인
6. 사이트에서 Ctrl+Shift+R 후 영상 재생 + 음성 재생 테스트
