# KDrama TOPIK 자막 설명 생성 프롬프트

## 프로젝트 개요
네팔 노동자를 위한 한국어 학습 앱 "KDrama TOPIK"의 자막별 문법/단어 설명을 생성하는 작업입니다. 사랑의 불시착 드라마 클립의 한국어 자막을 한 문장씩 분석하여 KR(한국어 설명)과 NP(네팔어 설명)을 만듭니다.

## 핵심 규칙

### 1. TTS 음성 생성용 텍스트
- 이 텍스트는 ElevenLabs에서 음성으로 변환됩니다
- ElevenLabs는 괄호() 안의 내용을 읽지 않습니다
- 이 특성을 활용하여 KR/NP 설명 모두 괄호 안에 반대 언어를 1:1 병기합니다
- 슬래시/, 등호=, 화살표→ 등 다른 기호는 사용 금지
- 앱에서 음성 재생 시 텍스트가 함께 스크롤됩니다

### KR 한국어 설명의 병기 규칙
- ElevenLabs가 데바나가리 괄호를 읽는 오류가 발생하므로, KR 설명에는 괄호 병기를 하지 않습니다
- 대신 본문 설명에서 네팔어 대응어를 자연스럽게 문장으로 풀어 설명합니다
- 예: 잠시는 아주 짧은 시간을 뜻합니다. 네팔어의 케히 베르에 해당합니다.

### NP 네팔어 설명의 병기 규칙
- 데바나가리 발음 뒤에 괄호로 한글 원문을 병기
- 예: चाम्सि हु (잠시 후) भनेको एकदमै छोटो समय हो।
- 예: ओजोन (오전) भनेको बिहानको समय हो।
- TTS는 데바나가리만 읽고, 화면에서는 한글도 보여서 학습에 도움

### 2. ElevenLabs 문자수 제한 (필수)
- ElevenLabs v3 모델은 표기상 5,000자 제한이지만, 데바나가리 문자는 내부 토큰을 더 소비하여 약 3,500자에서 음성이 끊김
- 따라서 다음 제한을 반드시 지킵니다:
  - **NP (네팔어 설명): Python len() 기준 3,500자 이내**
  - **KR (한국어 설명): Python len() 기준 3,000자 이내**
- KR과 NP는 1:1 내용 매칭이 필수이므로, 한쪽을 줄이면 다른 쪽도 같이 줄여야 합니다
- 문자수를 줄이기 위한 압축 전략:
  1. **같은 말 반복 삭제**: "네팔어의 ~에 해당합니다" 다음에 "네팔어로 ~와 같습니다" 같은 중복 제거
  2. **이전 자막에서 이미 설명한 단어는 간단히**: 예를 들어 들(복수), 에(조사) 등이 이미 앞에서 상세히 나왔으면 "들은 복수를 나타내는 말입니다" 한 줄로 축소
  3. **예문 수 조절**: 핵심 예문 1~2개만 유지, 비슷한 예문 여러 개 나열 금지
  4. **네팔어 비유 중 덜 중요한 것 생략**: 핵심 대응어만 유지
  5. **설명이 3문장 이상 늘어진 포인트는 1~2문장으로 압축**
- 이 제한은 품질이 아니라 분량만 줄이는 것이며, 핵심 규칙(단어 뜻, 네팔어 대응, 북한→표준어, TOPIK 연계, 문화 포인트)은 반드시 유지합니다

### 3. TTS 숫자/시간 표기 원칙
- 숫자를 그대로 쓰면 TTS가 한자어로 읽어버립니다 (예: 7시 → "칠시"). 실제 한국어 발음과 다릅니다.
- 숫자는 반드시 실제 발화 기준으로 한글로 풀어서 표기합니다.
- 괄호로 숫자 병기 금지. TTS가 괄호까지 읽습니다.

시간 읽기 (고유어):
- 1시 → 한 시, 2시 → 두 시, 3시 → 세 시, 4시 → 네 시
- 5시 → 다섯 시, 6시 → 여섯 시, 7시 → 일곱 시, 8시 → 여덟 시
- 9시 → 아홉 시, 10시 → 열 시, 11시 → 열한 시, 12시 → 열두 시

분/초 읽기 (한자어):
- 30분 → 삼십 분, 15초 → 십오 초

개수 읽기 (고유어):
- 1개 → 한 개, 2개 → 두 개, 3명 → 세 명

날짜/월 읽기 (한자어):
- 3월 → 삼월, 15일 → 십오일

금액/번호 읽기 (한자어):
- 1번 → 일 번, 100원 → 백 원

핵심: 한국 사람이 실제로 입으로 말하는 그대로 한글로 적는다.

### 4. 설명 깊이와 스타일
- 단어 하나하나 분해하여 설명
- 각 단어마다 네팔어 대응어 제시 (네팔어 발음을 한글로 자연스럽게 붙여 표기)
- 문법 패턴은 반드시 예문과 함께 설명
- 문화 포인트, 드라마 맥락 포함
- 존댓말/반말 차이, 격식체/비격식체 차이 설명
- 사투리, 북한말, 은어, 신조어가 나오면 특히 자세하게 풀이

### 5. 네팔어 발음 표기 원칙 (KR 설명에서)
- KR 한국어 설명에서 네팔어 대응어를 쓸 때, 네팔어 발음을 한글로 자연스럽게 붙여 표기
- ❌ 끊어 읽기: "잠-시 후" → "जाम-सि हु"
- ✅ 붙여 읽기: "잠시 후" → "जाम्सी हु"
- 실제 한국어 발화 기준, 연음/축약 반영

### 6. 한국어 단어 데바나가리 표기 원칙 (NP 설명에서)
- NP 네팔어 설명에서는 한국어 단어를 데바나가리 문자로 발음 표기해야 합니다
- 네팔 학습자는 한글을 읽지 못하므로, 자기네 문자인 데바나가리로 한국어 발음을 써줘야 어떤 단어인지 알 수 있습니다
- 하이픈 없이 자연스럽게 붙여 씁니다. TTS가 하이픈에서 끊어 읽기 때문입니다
- ❌ चाम-सि हु (하이픈 → TTS 끊김)
- ✅ चाम्सि हु (붙여쓰기 → 자연스러움)
- 실제 한국어 발화 기준으로 가장 가까운 데바나가리 표기를 사용합니다
- NP 첫줄(원문 데바나가리 표기)은 쉼표(,)로 의미 단위를 끊어줍니다
  - ElevenLabs TTS가 데바나가리 공백을 무시하고 붙여 읽는 경향이 있음
  - 쉼표를 넣으면 TTS가 자연스럽게 끊어 읽음
  - ❌ "उरि गुनुइ आन्बोरुल् उइह्योप्पानुन् कु ओट्टोन् हेङउइरादो ने नुने ट्टिनुन् नारेनुन् कुनाल्लो क्कुछिया." (한덩어리로 읽힘)
  - ✅ "उरि गुनुइ आन्बोरुल् उइह्योप्पानुन्, कु ओट्टोन् हेङउइरादो, ने नुने ट्टिनुन् नारेनुन्, कुनाल्लो क्कुछिया." (자연스럽게 끊김)
  - 한국어 원문의 호흡 단위에 맞춰 쉼표 배치
- ㄹ받침 + ㄹ초성 (ㄹㄹ 연음)은 ल्ल 로 표기합니다
  - 물량 → मुल्ल्यांग (ㄹ+ㄹ = ल्ल)
  - 설립 → सोल्लिप
  - 일류 → इल्ल्यु
- ㅇ받침 (ㅇ종성)은 ङ 으로 표기합니다
  - 항의 → हाङइ
  - 왕 → वाङ
  - 팀장 → टिमजाङ
- ㅂ받침 + ㄷ초성은 प्त 로 표기합니다 (경음화)
  - 아깝다 → अक्काप्ता
  - 좁다 → चोप्ता
  - 잡다 → चाप्ता
- ㅟ(위) 모음은 वि 가 아닌 उइ 로 표기합니다
  - वि = "비"로 들리므로 부정확
  - उइ = "우이" → 네팔 사람 귀에 "위"로 들림
  - 대위 → देउइ (❌ देवि → 네팔어 "여신"과 혼동)
  - 소위 → सोउइ (❌ सोवि)
  - 중위 → चुङउइ (❌ चुङवि)
  - 위험 → उइहोम (❌ विहोम)
- 받침(종성)이 있는 글자는 반드시 할(्)을 붙여서 순수 자음으로 끝내야 합니다
  - 할(्)이 없으면 데바나가리는 자동으로 "아" 모음을 붙여 읽습니다
  - ❌ कोत (것 → "코타"처럼 들림)
  - ✅ कोत् (것 → "콧"으로 정확히 끊김)
  - 예: 것 같은데 → कोत् कातुन्दे
  - 예: 먹다 → मोक्दा (ㄱ받침에 할 적용)

예시:
- 잠시 후 → चाम्सि हु
- 오전 → ओजोन
- 기사 → किसा
- 예정입니다 → येजोङइम्निदा
- 업데이트 할 거야 → अपडेट हाल कोया
- 물량 → मुल्ल्यांग
- 항의 → हाङइ

### 7. 동음이의어/다의어 전수 설명
단어를 설명할 때, 그 단어에 여러 뜻이 있으면 반드시 모든 뜻을 나열한 뒤 "여기서는 ~번째 뜻"이라고 짚어줘야 합니다. 네팔 학습자는 한국어 단어 하나가 여러 뜻을 가진다는 것을 모르기 때문에, 처음 만나는 단어마다 동음이의어를 전부 풀어줘야 혼동 없이 배울 수 있습니다.

예시:
- 배: 첫째, 먹는 과일 배가 있습니다. 둘째, 바다 위에 타는 배가 있습니다. 셋째, 몸속에서 음식을 소화하는 부분인 배가 있습니다. 여기서 '배가 고프다'의 배는 세 번째 뜻으로, 위장이 비어서 음식을 먹고 싶다는 뜻입니다.
- 기사: 첫째, 신문이나 뉴스에 실리는 글인 뉴스 기사가 있습니다. 둘째, 택시나 버스를 운전하는 사람인 운전기사가 있습니다. 셋째, 중세 유럽의 무사를 뜻하는 기사가 있습니다. 여기서는 첫 번째 뜻인 뉴스 기사입니다.

### 8. 사투리/북한말 설명 (해당되는 경우)
- 표준어 대응 제시
- 발음 변화 패턴 설명
- 어미 변화 분석
- 남한에서는 이렇게 말한다는 비교
- 네팔어 비유가 가능하면 활용

### 9. TOPIK 시험 연계 설명
- 드라마 대사에 나온 단어/문법 중 TOPIK 시험에 자주 나오는 것이 있으면 반드시 짚어줍니다
- "토픽 시험에서 자주 나오는 표현입니다" 또는 "토픽 시험에서 이 조사가 자주 출제됩니다" 같은 문구를 넣습니다
- 북한식 표현이 나오면 반드시 남한 표준어로 바꾼 문장을 함께 제시합니다 (TOPIK은 표준어 기준이므로)
- 예문도 TOPIK에 나올 법한 일상 표현으로 추가합니다
- 예시:
  - 까지는 토픽 시험에서 자주 나오는 조사입니다. 시험까지 공부해야 해, 밤까지 일했어, 이런 식으로 씁니다.
  - 이 문장을 남한 표준어로 바꾸면 "안됐구만"이 됩니다. 딱하게 됐구먼은 북한식 표현이고, 남한에서는 안됐구만 또는 안됐네라고 합니다.
  - 돌개바람은 남한에서 회오리바람이라고도 합니다. 토픽 시험에서는 회오리바람이라는 표현이 나올 수 있으니 함께 기억하면 좋습니다.

### 10. 문장 구조
각 자막 설명은 다음 형식을 따릅니다:

```
[자막 번호와 원문 한국어 문장]
이 문장은 [장면 맥락 설명]입니다. 하나씩 살펴보겠습니다.

첫째, [단어]. [단어 뜻 설명]. 네팔어의 "[네팔어 대응]"와 같습니다. [추가 설명이나 예문].

둘째, [단어/표현]. [설명]. [예문]. [문법 포인트].

...

마지막으로 문화 포인트입니다. [드라마 맥락이나 한국 문화 설명]. (해당되는 경우에만)
```

## 설명 예시

### 입력
자막: "왜 리정혁 중대장만 이렇게 특별합니까?"

### 출력 (KR 한국어 설명)
이 문장은 마을 아주머니가 리정혁에게 특별 대우를 하는 것에 대해 묻는 장면입니다. 하나씩 살펴보겠습니다.

첫째, 왜. 왜는 이유를 물어보는 말입니다. 네팔어의 "끼나"와 같습니다. 끼나 야쏘 가르누훈차? 왜 이렇게 하세요? 이런 식으로 문장 앞에 와서 이유를 묻습니다.

둘째, 리정혁 중대장. 중대장은 군대에서 부대를 이끄는 사람입니다. 여기서 중요한 것은 한국어에서는 이름 뒤에 직책을 붙인다는 것입니다. 리정혁 중대장, 김 선생님, 박 대리님, 이렇게 씁니다. 네팔어에서는 이름 앞에 존칭을 붙이지만, 한국어는 반대로 뒤에 붙입니다. 이것이 큰 차이입니다.

셋째, 만. 이것은 한국어에서 아주 자주 쓰이는 중요한 말입니다. 네팔어의 "마뜨라"와 똑같습니다. 리정혁만, 이것은 리정혁 마뜨라라는 뜻입니다. 다른 사람은 해당이 안 되고, 오직 리정혁 한 사람만이라는 뜻이에요. 예를 들어, 나만 알아. 이것은 네팔어로 말라이 마뜨라 타하 차. 이렇게 쓰입니다. 또 하나 예를 들면, 너만 좋아해. 이것은 띠미라이 마뜨라 만 빠라우추. 이렇게 됩니다.

넷째, 이렇게. 이렇게는 정도나 방법을 강조할 때 쓰는 말입니다. 네팔어로는 야띠, 또는 야사리에 해당합니다. 이 문장에서는 놀라움이 담겨 있어서, 이렇게까지 특별하단 말이야? 라는 느낌입니다.

다섯째, 특별합니까. 이 부분이 한국어를 배울 때 가장 중요한 포인트입니다. 원래 단어는 특별하다입니다. 이것을 높은 존댓말로 말하면 특별합니다가 됩니다. 그리고 질문으로 바꾸면 특별합니까가 됩니다. 여기서 중요한 점은, 합니다와 합니까는 격식을 차려야 하는 공식적인 상황에서 쓰는 말투라는 것입니다. 군대, 회사, 뉴스 같은 곳에서 씁니다. 친구 사이에서는 특별해? 라고 편하게 말합니다. 네팔어에서도 비슷합니다. 비쉐쉬 훈차, 이것은 특별합니다. 비쉐쉬 훈차? 물음표를 붙이면 특별합니까가 됩니다. 한국어에서는 말할 때는 합니다, 물어볼 때는 합니까로 끝이 바뀝니다. 이 패턴은 모든 동사에 적용됩니다. 예를 들어, 먹습니다, 먹습니까. 갑니다, 갑니까. 이렇게 말할 때와 물어볼 때 끝이 달라집니다.

마지막으로 문화 포인트입니다. 이 장면 바로 다음에 마을 사람들이 리정혁을 천연기념물이라고 부릅니다. 천연기념물은 나라에서 보호하는 귀중한 자연유산을 말합니다. 리정혁이 너무 잘생기고 집안도 좋으니까, 마을 전체가 나서서 보호해야 한다는 유머입니다. 이것이 이 장면의 가장 재미있는 부분입니다.

## KR 표시용 (Display) 규칙

### 개요
KR은 두 가지 버전을 만든다:
- **음성용 (TTS)**: 순수 한국어만. ElevenLabs가 읽을 용도. txt 파일로 저장.
- **표시용 (Display)**: 한국어 + 데바나가리 발음 + 네팔어 의미 병기. 화면에 보이는 용도. JSON kr 필드에 저장.

### 병기 형식
```
한국어 (데바나가리발음, 네팔어의미)
```
예시: 대기중에 (देगिजुङ्ए, तयारीमा पर्खनु)

네팔 학습자가 한국어 음성을 들으면서 화면을 볼 때:
- 데바나가리 발음 → "아 이렇게 발음하는구나"
- 네팔어 의미 → "아 이 뜻이구나"
- 반복 학습으로 같은 단어가 계속 나오면서 자연스럽게 습득

### 병기 규칙

1. **음성용 KR txt를 기준으로 작업** (절대 원본 변경 금지)
   - 사용자가 제공한 음성용 txt 파일이 원본
   - 한국어 문장, 문법, 단어, 내용 100% 그대로 유지
   - 오직 괄호 (발음, 의미) 만 추가

2. **NP 첫줄의 데바나가리(한글) 매핑을 역변환 + 네팔어 의미 추가**
   - NP 첫줄: देदेपोवि (대대보위) → KR 표시용: 대대보위 (देदेपोवि, ब्याटालियन सुरक्षा)
   - NP 첫줄: चिदोवोन् (지도원) → KR 표시용: 지도원 (चिदोवोन्, निर्देशक)

3. **본문에서도 NP에 나온 단어만 병기**
   - NP에서 데바나가리 (한글) 로 나온 단어 → KR에서 한글 (데바나가리, 네팔어의미) 로 병기
   - NP에 없는 단어는 절대 건드리지 않음
   - 같은 단어가 반복될 경우에도 매번 병기 (학습 효과)

4. **네팔어 설명도 병기**
   - 기존: 네팔어의 OOO에 해당합니다
   - 표시용: 네팔어의 OOO (데바나가리) 에 해당합니다

5. **본문 "네팔어의 ~" 부분: 한국어 음역 제거, 데바나가리만 표기**
   - KR 표시용에서 "네팔어의 한국어음역 (데바나가리)" 패턴 → "네팔어의 데바나가리" 로 변경
   - 한국어로 네팔어를 억지 표기하면 촌스럽고 딱딱한 기계어처럼 보임
   - 네팔 학습자는 데바나가리를 읽을 수 있으므로 한국어 음역 불필요
   - ❌ 네팔어의 케히 차이나 (केही छैन) 에 해당합니다
   - ✅ 네팔어의 केही छैन 에 해당합니다
   - ❌ 네팔어로 파니레 비쩨요 (पानीले भिज्यो) 와 같습니다
   - ✅ 네팔어로 पानीले भिज्यो 와 같습니다
   - 주의: 첫줄 괄호 (발음, 의미) 와 본문 괄호 (발음, 의미) 는 그대로 유지. 이 규칙은 오직 "네팔어의/로/에서" 뒤의 한국어 음역만 제거

6. **카타카나(일본어) 절대 사용 금지**
   - 데바나가리(네팔어)만 사용
   - NP에 데바나가리가 없는 단어는 괄호 자체를 넣지 않음

7. **숫자 표기 규칙**
   - 음성용 (TTS): 한글로 표기 (오중대장, 일, 이, 삼) → ElevenLabs가 정확히 읽음
   - 표시용 (Display): 아라비아 숫자로 표기 (5중대장, 1, 2, 3) → 눈으로 직관적
   - 모든 숫자에 적용: 일→1, 이→2, 삼→3, 사→4, 오→5 등
   - 제일→제1, 제이→제2, 제삼→제3, 제오→제5 등
   - 학습 효과: 화면에 "5"가 보이고 귀로 "오"가 들리면 "5를 오라고 읽는구나!" 자연스럽게 습득
   - 이 규칙은 원본 텍스트 변경 금지의 유일한 예외

### 저장 규칙
- JSON kr 필드 → KR 표시용 (병기 버전)
- JSON np 필드 → NP 표시용 (병기 버전)
- txt 파일 (_kr.txt) → KR 음성용 (순수 한국어) — TTS용
- txt 파일 (_kr_display.txt) → KR 표시용 (병기 버전) — 참고용
- txt 파일 (_np.txt) → NP 음성용 (순수 네팔어, 괄호 없음, 알파벳 그대로 유지) — TTS용
- txt 파일 (_np_display.txt) → NP 표시용 (병기 버전) — 참고용

### JSON 무결성 검증 (필수)
- 새 자막 설명을 JSON에 추가할 때 반드시 다음을 확인:
  1. **sub_id 일치**: explanations[N]의 sub_id는 반드시 subtitles[N]의 id와 같아야 함
     - subtitles의 id가 순차적이 아닐 수 있음 (중간에 빠진 번호 있음)
     - 절대로 단순히 index+1로 맞추면 안 됨!
     - sub_id가 틀리면 앱에서 음성 파일을 못 찾아 버튼 비활성화되거나 다른 번호 설명이 나옴
  2. **audio_kr / audio_np 키 필수**: 모든 explanation에 반드시 포함
     - 형식: "E01-C02_28_kr.mp3", "E01-C02_28_np.mp3"
  3. **키 순서 일관성**: sub_id, kr, np, audio_kr, audio_np 순서 유지
- JSON 수정 후 반드시 전체 검증 스크립트 실행:
  ```python
  for idx in range(min(len(d['subtitles']), len(d['explanations']))):
      assert d['subtitles'][idx]['id'] == d['explanations'][idx]['sub_id'], \
          f"[{idx}] subtitle.id={d['subtitles'][idx]['id']} != sub_id={d['explanations'][idx]['sub_id']}"
  ```

### 한 자막당 4개 버전
1. KR 음성용 (_kr.txt) — 순수 한국어, ElevenLabs TTS용
2. KR 표시용 (_kr_display.txt, JSON kr) — 한국어 (발음, 의미) 병기
3. NP 음성용 (_np.txt) — 순수 네팔어, 괄호 없음, 알파벳 그대로 유지, TTS용
4. NP 표시용 (_np_display.txt, JSON np) — 네팔어 (한글) 병기

### NP 음성용 규칙
- 기존 NP 표시용에서 괄호 (한글) 부분만 제거
- 알파벳(영어)은 그대로 유지 (ElevenLabs가 잘 읽음)
- 예시: 표시용 "टोपिक् (TOPIK) परीक्षामा" → 음성용 "TOPIK परीक्षामा"

### NP 표시용 숫자 규칙
- NP 괄호 안에 한국어 숫자가 나올 때 아라비아 숫자를 /로 병기
- (일) → (일/1), (이) → (이/2), (삼) → (삼/3), (오) → (오/5) 등
- 주의: 숫자 문맥에서만 적용. 사람 이름의 "이"(이씨 성) 등 숫자가 아닌 경우는 절대 /숫자 붙이지 않음
- 학습 효과: 네팔 사람이 한글 숫자와 아라비아 숫자를 동시에 볼 수 있음

### 예시
음성용: 토대는 북한에서 출신 성분, 가족 배경을 뜻하는 말입니다.
표시용: 토대 (तोदे, 사회적 배경) 는 북한에서 출신 성분, 가족 배경을 뜻하는 말입니다.

### KR 작성 스타일 규칙

1. **문법 용어 금지 — 쉬운 말로 풀어쓰기**
   - 네팔 노동자가 "구어체", "일인칭", "조사", "관형사형", "인용" 같은 문법 용어를 이해하기 어려움
   - ❌ 나는 일인칭 반말 표현입니다
   - ✅ 나는 자기 자신을 가리키는 말입니다. 반말로 쓰는 표현입니다.
   - ❌ 거는 것의 구어체이고
   - ✅ "거"는 "것"과 같은 뜻입니다
   - ❌ 다고는 인용을 나타내는 조사로
   - ✅ "다고"는 남의 말이나 생각을 전할 때 쓰는 표현입니다
   - 단, "반말"은 프로젝트 핵심 개념이므로 계속 사용 OK

2. **따옴표("")로 핵심 단어/표현 강조**
   - 단어를 설명할 때, 뜻풀이할 때 따옴표로 감싸기
   - TTS는 따옴표를 읽지 않으므로 음성에 영향 없음
   - 화면에서 눈에 잘 들어옴
   - 예: "조철강이는"에서 "이"는 이름 뒤에 붙는 말입니다
   - 예: "높다고"는 "높다고 해서", "높다는 이유로"라는 뜻입니다

3. **네팔어 대응어 교차 검증**
   - KR에 네팔어 대응어를 넣을 때 Gemini 등으로 교차 검증 권장
   - 예: "봐주다" = छुट दिनु (할인/면제 느낌) ❌ → छोड्ने छैन (눈감아주지 않겠다) ✅
   - 드라마 맥락에 맞는 네팔어 표현 선택이 중요
